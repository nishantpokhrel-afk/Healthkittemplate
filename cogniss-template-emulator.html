Setup JSON schema

{ "user": { "displayName": "...", "xp": 0, "coins": 0, ... }, "activity": { "engine": "DEFAULT", "randomizeQuestions": false, "questionLimit": null, "textToSpeech": false }, "templateVariation": { }, "questions": [ { "text": "...", "type": "MULTICHOICE", "options": [ { "text":"...", "correct":true, "score":10 } ], "duration": 20 } ], "tables": [ { "id": "table_x", "name": "Reward Tiers", "columns": [ { "displayName":"tierName", "valueType":"string" } ], "rows": [ { "tierName":"Bronze" } ] } ], "theme": { "colors": { "primary":"#5fb3a3" }, "rounded":"MEDIUM" }, "health": { "days": 14, "stepsRange": [3000,13000], "rhrRange": [54,72], "weightStart": 76, "workoutsCount": 6, "noData": false } } All keys optional — only what you include gets applied; everything else keeps its current value. Option objects only need "text"/"correct"/"score" — IDs are generated for you.

Updated automatically via REST_REQUEST (put) and on QUIT/EXIT.
Images resolve automatically — any File ID a template requests that you haven't uploaded yourself gets a real random photo from Picsum Photos (a free, no-key-required internet image library), sized sensibly if the ID mentions "icon", "background"/"banner"/"header", so nothing ever renders as an empty box. Upload here only when you need a specific image, audio, or video file. Each upload becomes a mock File record — when a template calls REST_REQUEST('get','File', id) with a matching ID, the emulator returns your uploaded asset instead. Copy the ID and paste it wherever a template expects a File ID (a question's image/audio field, a Table's objectid column, a theme background, etc.) — or use the quick-assign buttons for common theme slots.
